 学習記録システム（アプリアイデンティティ名があればここに）

Spring Bootとデータベース（DB）を用いた、日々の学習内容や成果物を一元管理・記録するためのシステムです。

　制作目的

1. ポートフォリオや学習資産の増加に伴う管理課題の解決
今後、様々な技術スタックでポートフォリオを制作していくにあたり、以下の課題を予測しました。
・何をどこにどのような意図で作ったのかの管理が煩雑になる
・ 過去の自分の学習プロセスや実装コードを振り返る際、検索性が低い

この課題を解決するため、自身の学習記録や成果物のメタデータを一元管理し、いつでも瞬時に振り返ることができるシステムを自作することにしました。

2. 技術的挑戦（Spring Boot × RDBの習得）
バックエンドエンジニアとしてのスキルアップのため、実務での採用実績が豊富なSpring Bootおよびリレーショナルデータベース（RDB）を用いたWebアプリケーション開発に挑戦することを目的としています。
データの永続化（CRUD処理）の基礎を体系的に学ぶための実践の場として制作しました。

　使用技術（テックスタック）

・バックエンド Java / Spring Boot (Spring MVC, Spring Data JPA)
・フロントエンド HTML / CSS / Thymeleaf 
・データベース H2 

 主な機能
・学習記録 / 成果物のCRUD機能: 作成したアプリの概要、URL、使用技術、学習したことなどの登録・編集・削除

 こだわったポイント / 工夫した点
・将来の自分というユーザーの目線に立った設計 : 今後ポートフォリオが増えても迷子にならないよう、直感的に整理・検索できるUI/UXを意識しました。
・Spring Bootによる堅牢なバックエンド実装 : コントローラー、サービス、リポジトリのレイヤーを適切に分離し、保守性の高いコードを意識しました。
